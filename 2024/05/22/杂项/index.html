正则表达式 '<[^>]+>' 用于匹配 HTML 标签。我们来逐步解释这个表达式的每个部分：

**<**：匹配 HTML 标签的开始符 <。

**[^>]+**：匹配一个或多个不是 > 的字符，也就是标签内容部分。

**>**：匹配 HTML 标签的结束符 >。

**<**：

  • 这个符号直接匹配 HTML 标签的起始字符 <。

**[^>]+**：

  • **[ 和 ]**：表示字符集，也就是在方括号内定义的一组字符。字符集中包括的字符会被匹配。
  • **^**：在字符集的开头，表示取反，即不包含字符集中的字符。
  • **>**：这是字符集中唯一的字符。因为 ^ 取反的作用，所以 [^>] 表示匹配任何不是 > 的字符。
  • **+**：表示前面的字符（在这里是 [^>]）至少出现一次，或多次。换句话说，匹配一个或多个不包含 > 的字符。

**>**：

  • 这个符号直接匹配 HTML 标签的结束字符 >。

session和requests

requests库是Python中用于发送HTTP请求的库，而Session对象是该库提供的一种高级功能。以下是两者的区别和用途：

requests

  • 功能：requests库的基础功能是发送HTTP请求，包括GET、POST、PUT、DELETE等请求。

  • 特点：每次发送请求时，都会重新建立一个新的连接。这意味着，每次请求都不会共享任何状态（如Cookie、会话等），即使是对同一个服务器的多个请求。

  • 用法：

Session

  • 功能：Session对象是requests库中提供的一个高级功能，用于在多个请求之间共享一些参数和信息。它可以管理和维护会话状态。

  • 特点：

    • 持久化Cookie：在使用Session对象发送多个请求时，可以自动管理和共享Cookie。比如，登录一次后，后续请求会自动携带登录后的Cookie。
    • 连接复用：Session对象会复用连接，减少重复建立连接的开销，提高性能。
    • 共享配置：可以在Session对象中设置一些默认参数，如headers、认证信息等，这些参数会自动应用到该会话中的所有请求。

  • 用法：

主要区别

  1. 状态管理：

    • requests：每次请求都是独立的，不会共享状态。
    • Session：可以在多个请求之间共享状态，如Cookie、headers等。

  2. 性能：

    • requests：每次请求都会重新建立连接。
    • Session：会复用连接，性能更高。

  3. 用例：

    • requests：适用于一次性请求，或不需要共享状态的请求。
    • Session：适用于需要在多个请求之间共享状态的场景，如登录后连续访问多个页面。
