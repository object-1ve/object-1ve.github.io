c#的基本架构

在C#中，通常使用一种称为”三层架构”或”三层架构模式”的设计模式来组织代码。这种模式将应用程序分为三个主要部分：业务逻辑层（BLL），数据访问层（DAL），和用户界面层（通常是UI层或Presentation层）。有时还会包括一个模型层（Model层）来表示应用程序的数据模型。这些层次的主要区别在于其职责和功能：

  1. 业务逻辑层（BLL）：BLL负责实现应用程序的业务逻辑。这包括处理数据验证、计算和应用业务规则等。BLL通常是应用程序中最复杂的部分，因为它需要理解并实现应用程序的核心业务逻辑。
  2. 数据访问层（DAL）：DAL负责与数据存储（通常是数据库）进行交互。它包含了与数据库通信的代码，负责执行查询、更新、插入和删除等操作。DAL的设计旨在将数据库操作与业务逻辑分离，使得修改数据库结构或更换数据库更加容易。
  3. 用户界面层（UI）：用户界面层负责向用户展示信息，并接受用户输入。它通常包含了图形用户界面（GUI）或命令行界面（CLI），以便用户与应用程序进行交互。UI层的代码通常较为简单，主要负责界面的布局和事件处理。
  4. 模型层（Model）：模型层用于表示应用程序的数据结构和业务对象。它通常包含了与数据存储无关的业务对象，这些对象被BLL和UI层共同使用。模型层有助于将数据结构与业务逻辑分离，提高代码的可维护性和可测试性。

总的来说，三层架构模式旨在将应用程序的不同部分分离开来，使得每一部分都可以独立开发、测试和维护。这种模式有助于提高代码的可扩展性、可重用性和可维护性，是许多现代软件开发项目中常用的设计模式之一。

Execute的区别

  • ExecuteNonQuery(): 用于执行不返回任何结果集的SQL语句，例如INSERT、UPDATE、DELETE等操作。它返回受影响的行数，通常用于执行非查询语句。
  • ExecuteReader(CommandBehavior.CloseConnection): 用于执行查询语句并返回一个SqlDataReader对象，该对象提供了对查询结果的只读、向前的访问。CommandBehavior.CloseConnection 参数指示在关闭 SqlDataReader 对象时关闭关联的连接。
  • ExecuteScalar(): 用于执行一个查询，并返回结果集的第一行第一列的值。通常用于执行返回单个值的查询，如返回一个聚合函数的结果或一个计算字段的值。

奇奇怪怪的错误:

在你的代码中，结束日期相等时不输出数据的原因可能是因为在查询时没有包含结束日期的当天。在 SQL 查询中，<= 表示小于或等于，但是不包括当天的日期。如果你希望包含结束日期当天的数据，可以修改查询条件为 <= @day_2，然后在赋值参数时将结束日期增加一天，以包含当天的数据。
